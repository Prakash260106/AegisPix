üîê Encryption Standards

AES-256-GCM Encryption

AegisPix. uses the Advanced Encryption Standard (AES) with 256-bit keys in Galois/Counter Mode (GCM). This is the same encryption standard used by governments and military organizations worldwide.

  • Key Size: 256 bits (2^256 possible combinations)
  • Mode: GCM (provides both confidentiality and authenticity)
  • Initialization Vector: 96 bits, randomly generated per file
  • Authentication: Built-in integrity verification

üîë Password-Based Key Derivation

PBKDF2 with SHA-256

We use Password-Based Key Derivation Function 2 (PBKDF2) to convert your password into a secure encryption key.

  • Hash Function: SHA-256
  • Iterations: 100,000 (makes brute-force attacks computationally expensive)
  • Salt: 128 bits, randomly generated per file
  • Output: 256-bit encryption key

üåê Client-Side Processing

All encryption and decryption occurs in your browser using the Web Crypto API:

  • No Server Access: Your images never leave your device
  • Zero Data Transmission: Nothing is uploaded or downloaded
  • Local Processing: All operations happen in your browser's secure environment
  • Memory Only: Files are processed in memory and immediately discarded

üìÅ File Format Security

Custom .pwimg Format

Our proprietary file format ensures maximum security and compatibility:

  • Magic Bytes: 'PWIMG' identifier for format verification
  • Version Control: Format versioning for future updates
  • Metadata Protection: All components are authenticated
  • Corruption Detection: Built-in integrity checks

üîí Security Best Practices

Password Recommendations:

  • Use at least 12 characters with mixed case, numbers, and symbols
  • Avoid common words, names, or easily guessable patterns
  • Use a unique password for each important file
  • Consider using a password manager for complex passwords

File Handling:

  • Keep backup copies of important original files
  • Store encrypted files in secure locations
  • Use secure communication channels when sharing passwords
  • Verify file integrity after encryption/decryption
