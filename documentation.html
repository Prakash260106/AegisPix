üöÄ Getting Started

Web Interface

  1. Open AegisPix in your Browser
  2. Click "Select Image" and choose your image file
  3. Enter a strong password (minimum 6 characters)
  4. Optionally enter a custom filename
  5. Click "Encrypt & Download" to get your .pwimg file

Decrypting Images

  1. Open AegisPix in your browser
  2. Click "Select .pwimg File" and choose your encrypted file
  3. Enter the password used during encryption
  4. Click "Decrypt & View" to see your image
  5. Use "Download Image" or "Copy to Clipboard" as needed

Browser Requirements

  • Web Crypto API support: Chrome 37+, Firefox 34+, Safari 11+, Edge 79+
  • File API support: All modern browsers
  • Blob support: All modern browsers
  • Download API: All modern browsers

üîß API Reference

Web Crypto API Usage

AegisPix uses the following Web Crypto API methods:

Key Derivation (PBKDF2)

// Derive encryption key from password
const keyMaterial = await crypto.subtle.importKey(
    'raw',
    new TextEncoder().encode(password),
    { name: 'PBKDF2' },
    false,
    ['deriveBits']
);

const derivedBits = await crypto.subtle.deriveBits({
    name: 'PBKDF2',
    salt: salt,           // 16 bytes
    iterations: 100000,   // PBKDF2 iterations
    hash: 'SHA-256'
}, keyMaterial, 256);

AES-GCM Encryption

// Import key for AES-GCM
const key = await crypto.subtle.importKey(
    'raw',
    derivedBits,
    { name: 'AES-GCM' },
    false,
    ['encrypt']
);

// Encrypt the image data
const encryptedData = await crypto.subtle.encrypt({
    name: 'AES-GCM',
    iv: iv    // 12 bytes initialization vector
}, key, imageData);

AES-GCM Decryption

// Import key for AES-GCM decryption
const key = await crypto.subtle.importKey(
    'raw',
    derivedBits,
    { name: 'AES-GCM' },
    false,
    ['decrypt']
);

// Decrypt the image data
const decryptedData = await crypto.subtle.decrypt({
    name: 'AES-GCM',
    iv: iv    // Same IV used for encryption
}, key, encryptedData);

Error Handling

The application handles the following error conditions:

  • Invalid file format: Files not starting with 'PWIMG'
  • Unsupported version: Version numbers other than 1
  • Decryption failure: Wrong password or corrupted data
  • Browser compatibility: Missing Web Crypto API support

üìÅ File Format Specification

.pwimg File Structure

Encrypted files use a custom binary format:

0x00 - 0x04 Magic Bytes 'PWIMG' (5 bytes) File format identifier
0x05 Version 1 (1 byte) Format version number
0x06 - 0x15 Salt 16 bytes PBKDF2 salt (random)
0x16 - 0x21 IV 12 bytes AES-GCM initialization vector
0x22 - EOF Encrypted Data Variable AES-GCM encrypted image data

Technical Details

  • Total header size: 34 bytes (magic + version + salt + IV)
  • Encryption: AES-256-GCM with 96-bit IV
  • Authentication: GCM provides built-in integrity verification
  • Key derivation: PBKDF2-SHA256 with 100,000 iterations

Compatibility

The .pwimg format is designed to be:

  • Cross-platform: Works on Windows, macOS, Linux
  • Future-proof: Version field allows format updates
  • Secure: No metadata leakage, authenticated encryption
  • Efficient: Minimal overhead (34 bytes header)

üîß Troubleshooting

Common Issues

"Web Crypto API not supported"

Solution: Update your browser to the latest version. AegisPix requires Chrome 37+, Firefox 34+, Safari 11+, or Edge 79+.

"Invalid file format" error

Solution: Ensure you're selecting a valid .pwimg file created by AegisPix. The file must start with the 'PWIMG' magic bytes.

"Decryption failed" error

Solution: Verify you're using the correct password. Passwords are case-sensitive. If the file is corrupted, try re-encrypting the original image.

Large files won't encrypt

Solution: Browser memory limits apply. Try splitting large files or use the desktop application for bigger images.

Debug Information

To help troubleshoot issues, please provide:

  • Browser name and version
  • Operating system and version
  • Exact error message
  • Steps to reproduce the issue
